# KSB1 自动导出使用与介绍文档

这份文档分成两部分：

- 第一部分：给使用者看，讲怎么安装、怎么运行、出问题怎么排查
- 第二部分：给分享者看，讲怎么向同事介绍这个工具

适用对象：

- 想自己使用 KSB1 自动导出工具的人
- 想把这个工具介绍给团队同事的人

## 一、这个工具是做什么的

`KSB1 自动导出工具` 的目标，是把每月重复的 SAP 导数动作自动化。

它主要解决的是这类问题：

- 每个月都要重复进入 SAP 做 KSB1 导出
- 人工步骤多，容易点错、漏点、选错月份
- 时间花在重复操作上，而不是花在结果核对和分析上

这个工具自动化的是：

- 打开流程
- 进入 KSB1
- 执行导出
- 输出结果文件

这个工具**不替代**：

- 财务判断
- 结果核对
- 异常分析

一句话介绍：

`它不是替你判断数据，而是替你完成重复的 SAP 导出动作。`

## 三、安装前准备

在使用这个工具之前，请先确认下面几项：

### 1. 已安装 SAP GUI

这是最关键的前提。没有 SAP GUI，工具无法执行导出。

### 2. SAP 账号可用

需要有正常可登录的 SAP 账号和对应权限。

### 3. VPN 可连接

如果业务环境依赖 VPN，那使用前要先连上 VPN。

### 4. Mac 系统权限已放开

首次运行时，macOS 可能会提示安全权限、磁盘访问权限或辅助功能权限。

如果被系统拦住，按提示允许即可。

### 5. 输出目录可写

工具运行后要写文件，所以输出路径必须可写。

### 第一次打开的注意事项

第一次在 macOS 上打开，可能遇到系统拦截。

处理方式：

1. 右键应用
2. 选择“打开”
3. 再确认一次

如果系统还是不放行，就去：

- `系统设置 -> 隐私与安全性`

把相关权限放开。

## 五、如何使用

### 使用步骤

打开应用后，按这个顺序操作：

1. 输入 SAP 用户名
2. 输入 SAP 密码
3. 选择年份
4. 选择月份
5. 选择输出目录
6. 点击开始

工具会自动：

- 连接流程
- 拉起 SAP
- 进入 KSB1
- 执行导出
- 生成结果

运行结束后，会把结果输出到指定文件夹。

### 建议使用方式

如果你是第一次用，建议：

1. 先选一个你熟悉的月份
2. 先跑一次测试
3. 先确认输出结果格式正确
4. 再作为正式工作流程使用

## 六、常见问题

### 1. 打不开应用

常见原因：

- 被 macOS 安全策略拦截
- 权限没放开
- 应用没有正确解压

优先处理方式：

- 右键打开
- 到系统设置里允许

### 2. 运行后没有反应

先检查：

- SAP GUI 是否已安装
- VPN 是否已连接
- 账号密码是否正确
- 输出目录是否可写

### 3. 能打开，但导出失败

常见原因：

- SAP 环境异常
- VPN 不稳定
- SAP 页面状态不对
- 权限不足

这个工具可以减少重复动作，但它不能绕过环境问题。

### 4. 别人的电脑上为什么跑不起来

最常见的原因不是工具本身，而是环境没准备好。

优先排查：

- 是否安装 SAP GUI
- 是否有权限
- 是否连上 VPN
- 是否放开系统权限

## 七、适合怎么给别人介绍

向同事介绍时，建议不要一开始就讲技术实现。

最好的顺序是：

1. 先讲痛点
2. 再讲工具解决什么问题
3. 再讲怎么用
4. 最后讲前提条件和边界

### 你可以这样介绍

`这个工具的作用，是把每个月重复的 KSB1 导出动作自动化。`

`以前要人工打开 SAP、进入事务、设置条件、导文件。现在工具可以把这套重复动作自动执行，减少点错和漏点。`

`使用者只需要输入账号密码、选择月份、点击开始，最后去看输出结果。`

`它自动化的是导出流程，不替代财务判断。`

## 八、给同事演示时怎么讲

### 1. 先讲价值

可以这样说：

`这个工具主要节省的是重复操作时间，减少的是人工点击错误。`

### 2. 再讲流程

给同事演示：

1. 打开应用
2. 填账号密码
3. 选年月
4. 点开始
5. 展示输出文件

### 3. 最后讲边界

要讲清楚：

- 需要 SAP GUI
- 需要权限
- 可能需要 VPN
- 不是所有机器拿到就一定能立刻跑

这样同事对工具的预期会更稳。
