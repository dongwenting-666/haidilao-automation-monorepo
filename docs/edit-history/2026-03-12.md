# 2026-03-12 — QBI Crawler library, output directory reorganization

## Summary

Added new `qbi-crawler` library for crawling Haidilao Quick BI dashboards via Playwright browser automation. Reorganized output directories so each automation writes to its own subfolder. Five rounds of code review cleaned up the library.

## Changes

### New Library: qbi-crawler

- **`auth.py`** — `QBISession` context manager: Playwright browser lifecycle, LDAP login via dynamically rendered form, auto-installs Chromium on first use (thread-safe with `threading.Lock` double-checked locking), password cleared after login, safe `__exit__` with nested try/finally
- **`dashboard.py`** — Dashboard navigation and export: direct URL navigation with menuIds (avoids iframe detachment from sidebar clicks), date range setting via keyboard input (Ant Design DatePicker workaround), XLSX export with Ant Design modal interaction
- **`constants.py`** — `BASE_URL = "https://qbi.superhi-tech.com"`
- **`errors.py`** — Exception hierarchy: `QBIError` (base), `QBILoginError`, `QBITimeoutError`
- **`__init__.py`** — Public API exports with `__version__` from importlib.metadata
- **`py.typed`** — PEP 561 type checker marker

### Reports Supported

### Output Directory Reorganization

- KSB1 CLI default output changed from `output/` to `output/ksb1/`
- KSB1 GUI default output changed from `output/` to `output/ksb1/`
- QBI crawler uses `output/qbi/` by convention (`DEFAULT_OUTPUT_SUBDIR = "qbi"`)

### Environment Variables

- Added `QBI_USERNAME` and `QBI_PASSWORD` to `.env.example`

### Code Review Fixes (5 rounds)

- Thread-safe browser install with double-checked locking pattern
- Safe `__exit__` resets all instance state to None
- Narrowed `except Exception` to `except OSError` in browser install
- Added `self._pw_context: Any` type annotation
- Extracted all inline timeout literals to named constants (`_SELECTOR_WAIT_TIMEOUT_MS`, `_EXPORT_BTN_TIMEOUT_MS`, `_DOWNLOAD_TIMEOUT_MS`)
- Removed unused `_MENU_ID` module variable
- Added `start_date <= end_date` validation in `download_report`
- Deduplicated `BASE_URL` into `constants.py`
- Null guards on all `wait_for_selector` results
