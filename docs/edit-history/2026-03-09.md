# 2026-03-09 — GUI project, LLM enhancement, PyInstaller EXE

## Summary

Added KSB1 Accounting Check GUI project with tkinter desktop app, PyInstaller packaging to standalone EXE, and LLM enhancement for rule-based findings. Multiple code review rounds cleaned up architecture.

## Changes

### New Project: ksb1-accounting-check-gui

- **`app.py`** — Full tkinter GUI: SAP credentials, month/year spinboxes, output directory picker, optional LLM model combobox (editable), action buttons, scrolling log area
- **`worker.py`** — Background worker functions (`run_download_and_generate`, `run_generate_only`) with user-friendly Chinese error messages for SAP/Ollama failures
- **`paths.py`** — Resource path resolution for PyInstaller frozen bundles vs dev mode. Uses `_find_repo_root()` with `@functools.cache` to walk up and find `pyproject.toml` with workspace marker
- **`log_handler.py`** — Thread-safe `QueueLogHandler` using `queue.Queue` + tkinter `after()` polling with `winfo_exists()` safety check
- **`ksb1_gui.spec`** — PyInstaller spec for single-file EXE (34 MB), bundles data files (报表科目.xlsx, cost_centers.txt, prompt.md), hidden imports for all workspace packages

### LLM Enhancement (hybrid rules + LLM)

- **`analyze.py`** — Added `model` and `prompt_path` parameters to `generate_report()`. When model is set, initializes LLM client and enhances findings after rule detection
- **`llm.py`** — Rewritten as finding-enhancer: `enhance_findings()` takes rule findings + transaction rows, sends to LLM for contextual explanation. Pre-computes grouped subtotals so LLM doesn't do arithmetic. Batching (`max_chars=4000`), retry logic, graceful fallback to rule-based observations
- **`prompt.md`** — Rewritten as enhancer prompt: receives findings + pre-computed subtotals, explains WHY differences exist
- **`llm.py`** — Added `set_prompt_path()` API to replace module-level monkey-patching

### Code Review Fixes (3 rounds)

- Replaced monkey-patching of `llm.PROMPT_FILE` with proper `set_prompt_path()` API and `prompt_path` parameter through call chain
- Fixed `callable` → `Callable[[bool, str], None]` type hints
- Replaced fragile 5-level `parent.parent...` with `_find_repo_root()` using workspace marker detection
- Added `winfo_exists()` guard in log handler polling
- Extracted `_common_kwargs()` in app.py and `_common_generate_kwargs()` in worker.py to reduce duplication
- Made model combobox editable for custom model names
- Fixed misleading `_simplify_name()` docstring
- Added `@functools.cache` to `_find_repo_root()`
- Inlined unused `month_spin`/`year_spin` variables

### PyInstaller Build

- Built standalone EXE: `dist/KSB1会计检查.exe` (34 MB)
- Fixed SPECPATH resolution in spec file (SPECPATH is a directory, not file path)
