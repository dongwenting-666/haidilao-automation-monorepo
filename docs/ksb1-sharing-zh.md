## 建议分享结构

### 1. 背景

可以这样开场：

`我们每个月做 KSB1 导出时，都要重复登录 SAP、进入事务、输入范围、导出文件。这个流程本身不复杂，但步骤多、重复性高，也容易出现漏点、点错、导错月份的问题。`

### 2. 工具解决的问题

重点讲这三个点：

- 减少人工重复操作
- 降低导出过程的操作错误
- 缩短每月固定动作的处理时间

可以直接说：

`这个工具不是替代大家判断数据，而是把机械性的导出过程自动化。`

### 3. 使用流程

建议现场展示这几个步骤：

1. 打开应用
2. 输入 SAP 账号密码
3. 选择月份和年份
4. 选择输出目录
5. 点击开始
6. 工具自动连接 SAP、执行 KSB1 导出、输出结果

### 4. 使用前提

这部分一定要提前讲，不然后面同事最容易问“为什么我这里跑不起来”。

当前前提条件：

- 本机已安装 SAP GUI
- 已具备可用的 SAP 账号权限
- 如果需要，VPN 已连接
- 本机系统权限已放开
- 首次运行应用时，macOS 可能会弹安全提示

### 5. 结果产出

讲清楚输出是什么：

- 自动导出的原始数据
- 生成后的结果文件
- 输出目录位置清晰可查

建议现场演示最后一步：

- 工具运行完成后，直接打开结果文件夹
- 给同事看最终产物，而不是只看运行过程

### 6. 工具边界

这一段很重要，会显得你讲得稳。

建议直接说：

- 工具自动化的是“导出流程”
- 它不替代财务判断
- 如果 SAP、VPN、权限或环境异常，流程仍然会失败
- 工具的价值是减少重复动作，不是把所有问题都自动解决

## 常见问题

### 1. 这个工具是不是替代人工核对？

不是。它替代的是重复导出动作，不替代财务判断。

### 2. 这个工具是不是所有人拿到都能直接用？

不是。前提是环境要准备好，比如 SAP GUI、权限、VPN、系统设置。

### 3. 如果 SAP 页面变化了怎么办？

那自动化流程可能需要调整，工具不是一劳永逸的。

### 4. 如果导出失败怎么办？

先检查 VPN、SAP 登录状态、权限和系统环境，再看日志定位原因。
