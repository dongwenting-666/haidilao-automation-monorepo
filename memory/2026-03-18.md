## VPN library overhaul (2026-03-18)

### Final working implementation: `libs/vpn/src/vpn/_darwin.py`

**Strategy**: quit CorpLink → launch fresh → click Overview VPN Connectivity
toggle → poll until connected → hide window (do NOT quit).

**Why quit first**: ensures predictable Overview tab state on fresh launch.

**Why Overview toggle (not Network tab Connect button)**:
The Network tab Connect button has non-deterministic React init time (3–40 s
after the tab mounts). The Overview VPN Connectivity toggle is immediately
interactive on fresh launch.

**Click tool: cliclick** (`brew install cliclick`).
- CGEvent without proper `CGEventSource` is silently ignored by Electron.
- AppleScript `click AXButton` unreliable (CorpLink doesn't expose buttons via AX API).
- `cliclick` uses `kCGEventSourceStateHIDSystemState` — Electron processes it correctly.

**Calibrated offsets** (window at AppleScript pos 100,100, size 900×560, CorpLink v3.1.21):
- `_OVERVIEW_TOGGLE_DX = 400`
- `_OVERVIEW_TOGGLE_DY = 115`

**Status detection**: tail `/usr/local/corplink/logs/corplink.log` (world-readable).
- `WireGuard Connected` — fires within ~1 s of tunnel coming up
- `reportVpnStatus start map[ip:…` — steady-state confirmation
- `vpn.go:\d+: VPN Disconnected` — tunnel down

**Session cycling**: `ensure_vpn(max_connected_hours=6.0)` reconnects if session
≥ 6 h, before the server-side 7.5-hour hard limit hits mid-automation.

**Hide not quit**: quitting sends `ClientClose` to `corplink-service` → WireGuard
tunnel torn down. Window is hidden; process stays running; tunnel stays alive.

**Tests**: 21 unit tests (`test_darwin.py`) + 5 e2e tests (`test_e2e.py`), all passing.
